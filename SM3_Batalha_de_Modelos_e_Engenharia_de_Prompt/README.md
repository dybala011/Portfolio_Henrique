# Batalha de Modelos & Engenharia de Prompt (XML) 🚀

Este projeto apresenta um estudo comparativo de diferentes modelos de Inteligência Artificial (LLMs) submetidos a um prompt estruturado em formato XML para a criação de uma página web.

## 📋 Sobre o Desafio
[cite_start]O objetivo principal foi construir um **Prompt Estruturado em XML** capaz de gerar uma página HTML5 Single Page com CSS3 integrado[cite: 8, 12, 13].

### Estrutura do Prompt Utilizada
[cite_start]O prompt seguiu um protocolo rigoroso de execução contendo[cite: 12, 26]:
* [cite_start]**Objetivo:** Criar página HTML5 única com CSS3 interno[cite: 13].
* [cite_start]**Diretrizes de Design:** Layout responsivo, minimalista e definições de tipografia[cite: 15, 16, 18].
* [cite_start]**Obrigatoriedades Técnicas:** Menu de navegação (âncoras), galeria/portfólio e rodapé de contato[cite: 19, 20, 21, 22].
* [cite_start]**Métrica de Saída:** Estimativa de tokens gerados ao final do código[cite: 24, 25].

## 📊 Análise Comparativa
[cite_start]Os testes foram realizados nos modelos: **ChatGPT, Gemini, DeepSeek, Qwen, Grok, Maritaca e Claude**[cite: 9, 28].

## 🧠 Reflexão Crítica
* [cite_start]**Maior Compreensão de XML:** O modelo **Gemini** demonstrou a melhor interpretação da estrutura proposta[cite: 95, 96].
* [cite_start]**Verbosidade vs. Qualidade:** Modelos com maior número de tokens nem sempre entregaram o melhor resultado; os tokens extras geralmente refletiam mais conteúdo, não necessariamente melhor qualidade[cite: 97, 98].
* [cite_start]**Veredito para Prototipagem:** * **Prototipagem Rápida:** Gemini[cite: 100].
    * [cite_start]**Códigos Complexos:** Claude[cite: 100].
